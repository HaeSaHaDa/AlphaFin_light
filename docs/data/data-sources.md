# AlphaFin LTE 데이터 소스 정의

## 문서 목적

이 문서는 AlphaFin LTE 프로젝트에서 사용하는
외부 데이터 소스를 정의한다.

목표는 다음과 같다.

- 데이터 소스 명확화
- 무료 데이터 기반 구조 유지
- 데이터 수집 범위 통일
- 데이터 한계 명확화
- AI 구현 방향 통일

현재 프로젝트는
무료 데이터 소스를 우선 사용한다.

---

# 현재 데이터 전략

현재 프로젝트는 다음 원칙을 따른다.

- 무료 데이터 우선
- 실험 가능한 수준 우선
- 한국 금융 데이터 우선
- 작은 규모부터 시작
- 재현 가능한 구조 유지

현재 단계에서는
대규모 상용 데이터는 사용하지 않는다.

---

# 현재 사용 데이터 종류

현재 프로젝트는 다음 데이터를 사용한다.

```text
1. 주가 데이터
2. 공시 데이터
3. 뉴스 데이터
4. 기업 기본 정보
```

---

# 주가 데이터

## pykrx

현재 프로젝트의
기본 주가 데이터 소스.

공식 문서:

```text
https://github.com/sharebook-kr/pykrx
```

수집 가능 데이터:

- KOSPI 시세
- KOSDAQ 시세
- OHLCV 데이터
- 거래량
- 시가총액
- 투자자별 거래
- ETF 데이터

예상 사용 목적:

- 가격 분석
- 시계열 데이터 저장
- Retrieval 보조 정보
- 간단한 백테스트

장점:

- 무료
- 한국 시장 특화
- 사용 단순
- pandas 친화적

한계:

- 실시간 데이터 제한
- API 안정성 제한 가능
- 대규모 요청 시 속도 제한 가능

예상 저장 위치:

```text
data/raw/price/
```

---

# 공시 데이터

## OpenDART

전자공시 데이터 수집 소스.

공식 사이트:

```text
https://opendart.fss.or.kr/
```

수집 가능 데이터:

- 사업보고서
- 분기보고서
- 반기보고서
- 주요 공시
- 기업 개요
- 재무 정보 일부

예상 사용 목적:

- RAG 문서 구성
- 기업 분석
- Retrieval 대상 생성
- 공시 기반 분석

장점:

- 무료
- 공식 데이터
- 한국 기업 대상
- 문서 기반 분석 가능

한계:

- 호출 제한 존재
- XML/JSON 구조 복잡
- 문서 정제 필요
- 긴 문서 처리 필요

예상 저장 위치:

```text
data/raw/dart/
```

---

# 뉴스 데이터

## 뉴스 크롤링

현재 프로젝트는
뉴스 API보다
기본 웹 크롤링 구조를 우선 고려한다.

예상 대상:

- 네이버 뉴스
- 한국경제
- 연합뉴스
- 매일경제

예상 사용 목적:

- 시장 이벤트 분석
- Retrieval 데이터 생성
- 뉴스 기반 Context 생성

예상 수집 정보:

- 제목
- 본문
- 날짜
- 언론사
- 종목 관련 정보

장점:

- 무료
- 데이터 확보 용이
- 다양한 뉴스 확보 가능

한계:

- robots.txt 확인 필요
- 사이트 구조 변경 가능
- HTML 정제 필요
- 품질 편차 존재

예상 저장 위치:

```text
data/raw/news/
```

# 데이터 저장 전략

현재 프로젝트는 다음 구조를 유지한다.

```text
data/
├─ raw/
├─ processed/
├─ chunks/
├─ embeddings/
└─ samples/
```

---

# Raw Data 원칙

Raw 데이터는
수집 직후 원본 상태를 유지한다.

원칙:

- 원본 수정 금지
- 원본 삭제 금지
- 재현 가능성 유지

---

# Processed Data 원칙

Processed 데이터는
전처리 완료 상태를 의미한다.

포함 가능 작업:

- 텍스트 정리
- HTML 제거
- Chunk 생성
- Metadata 생성

---

# 현재 DB 전략

현재 프로젝트는
MariaDB 기반 저장 구조를 우선 사용한다.

현재 목표:

- 로컬 개발 환경 구축
- 수집 데이터 저장
- 실험 구조 유지

향후 필요 시:

```text
PostgreSQL
또는
Supabase
```

전환 가능성을 열어둔다.

---

# 현재 제외 데이터

현재 사용하지 않는 데이터:

- 유료 Bloomberg 데이터
- Reuters 유료 데이터
- 실시간 초고속 체결 데이터
- 기관 전용 데이터
- Tick 단위 초고빈도 데이터

현재 프로젝트는
경량형 연구 구조를 목표로 한다.

---

# 현재 원칙

현재 프로젝트는 다음 원칙을 유지한다.

- 무료 데이터 우선
- 작은 규모 우선
- 재현 가능한 구조 유지
- AI 협업 가능한 구조 유지
- Retrieval 중심 데이터 구조 유지
- 과도한 인프라 사용 금지
