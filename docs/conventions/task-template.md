# AlphaFin LTE TASK 템플릿

## 문서 목적

이 문서는 AlphaFin LTE 프로젝트의
표준 TASK 구조를 정의한다.

목표는 다음과 같다.

- TASK 구조 통일
- AI 작업 흐름 통일
- Prompt 기록 구조 통일
- 작업 추적 가능성 확보
- 세션 복구 가능성 확보

현재 프로젝트의 모든 TASK는
이 템플릿 구조를 기준으로 작성한다.

---

# 기본 TASK 구조

```text
1. 상태
2. 목표
3. 범위
4. 제외 범위
5. 생성 대상
6. 구현 규칙
7. 검증 항목
8. 관련 Prompt
9. 관련 Logs
10. 완료 조건
11. 다음 TASK 후보

```

---

# 목표 작성 규칙

목표는:

- 짧고 명확하게 작성
- 하나의 목적만 가지기
- 현재 TASK 범위만 설명하기

현재 금지:

- 여러 목표 혼합
- 미래 기능 포함
- 추상적인 목표 작성

---

# 범위 작성 규칙

범위는:

- 현재 TASK에서 수행할 작업만 작성
- 구현 범위를 명확히 제한
- 작은 단위 유지

현재 금지:

- 범위 불명확 상태
- 대규모 구현 포함
- 관련 없는 기능 포함

---

# 제외 범위 작성 규칙

반드시 현재 TASK에서
구현하지 않을 내용을 명시한다.

목적:

- 범위 폭주 방지
- AI 오인식 방지
- 미래 기능 조기 구현 방지

---

# Prompt 기록 규칙

모든 실행 Prompt는
현재 TASK 기준으로
prompts/TASK-XXX/ 구조에 저장한다.

예시:

```text
prompts/TASK-001/prompt-001.md
```

TASK 문서는
작업 정의와 상태 관리만 담당한다.

---

# 결과 기록 규칙

모든 Prompt 실행 결과는
현재 TASK 기준으로
logs/TASK-XXX/ 구조에 기록한다.

포함 가능 내용:

- 생성 성공 여부
- 수정 내역
- markdown 오류
- 구조 오류
- 검증 결과

---

# 이슈 기록 규칙

문제 발생 시
반드시 기록한다.

포함 가능 내용:

- AI 오인식
- 구조 충돌
- markdown 오류
- directory 오류
- Prompt 충돌

---

# 검증 항목 규칙

모든 TASK는
검증 항목을 포함해야 한다.

최소 포함 항목:

- 실행 가능 여부
- markdown 정상 여부
- 구조 정상 여부
- 범위 준수 여부

---

# 완료 조건 규칙

완료 조건은:

- 명확하게 작성
- 검증 가능한 형태 유지
- 현재 TASK 범위 기준 유지

현재 금지:

- 추상적 완료 기준
- 검증 불가능한 완료 기준

---

# 현재 원칙

현재 프로젝트는 다음 원칙을 유지한다.

- 작은 TASK 유지
- Prompt 히스토리 유지
- 세션 복구 가능 구조 유지
- AI 작업 추적 가능 구조 유지
- 문서 우선 개발 유지

---

# 현재 금지 사항

현재 금지:

- Prompt 미기록
- 범위 없는 TASK
- 검증 없는 완료 처리
- TASK 외 구조 수정
- 대규모 기능 혼합
