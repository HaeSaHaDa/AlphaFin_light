# cursor-workflow.md

# AlphaFin LTE Cursor 작업 흐름

## 문서 목적

이 문서는 Cursor 기반 AI 작업 흐름을 정의한다.

목표는 다음과 같다.

- Cursor 작업 흐름 통일
- 문서 우선 개발 유지
- AI 오인식 최소화
- Prompt 단순화
- 세션 복구 가능성 확보

---

# 기본 작업 흐름

현재 프로젝트는
다음 순서로 작업한다.

```text
문서 확인
→ TASK 확인
→ 기존 Prompt 확인
→ Prompt 생성
→ 구현
→ 검증
→ Prompt 저장
→ 결과 기록
→ 이슈 기록
```

---

# 작업 시작 전 필수 확인

Cursor 작업 시작 전
반드시 다음 문서를 읽는다.

```text
1. README.md
2. AGENTS.md
3. 현재 TASK 문서
4. docs/project/*
5. docs/architecture/*
6. docs/conventions/*
7. 기존 prompts
8. 기존 logs
```

문서를 읽지 않고
바로 구현하지 않는다.

---

# 작업 단위 규칙

현재 프로젝트는
TASK 단위 작업을 기본으로 한다.

원칙:

- 하나의 TASK는 하나의 목적만 가진다.
- TASK 범위를 벗어나지 않는다.
- 관련 없는 수정 금지.
- 작은 단위 구현 유지.

---

# Prompt 사용 규칙

Prompt는
현재 작업 요청만 포함한다.

규칙 설명은
문서 시스템이 담당한다.

현재 프로젝트는:

```text
문서 = 규칙
Prompt = 작업 요청
```

구조를 유지한다.

---

# 구현 규칙

현재 프로젝트는 다음 원칙을 유지한다.

- 작은 함수 유지
- 작은 파일 유지
- 명시적 흐름 유지
- 과도한 abstraction 금지
- 구조적 추적 가능성 유지

---

# 검증 규칙

구현 후
반드시 검증한다.

최소 검증 항목:

- 실행 가능 여부
- markdown 정상 여부
- 구조 정상 여부
- 로그 생성 여부
- 예외 처리 여부

검증 없는 완료 처리 금지.

---

# Prompt 기록 규칙

모든 실행 Prompt는
현재 TASK 기준으로
prompts/TASK-XXX/ 구조에 저장한다.

예시:

```text
prompts/TASK-001/prompt-001.md
```

Prompt는
실제 실행 기록을 담당한다.

TASK 문서는
작업 정의와 상태 관리를 담당한다.

---

# 결과 기록 규칙

모든 작업 결과는
현재 TASK 기준으로
logs/TASK-XXX/ 구조에 기록한다.

예시:

```text
logs/TASK-001/result-001.md
```

포함 가능 내용:

- 생성 파일
- 수정 파일
- 오류 발생 여부
- 검증 결과
- 추가 수정 필요 여부

---

# 이슈 기록 규칙

문제 발생 시
반드시 기록한다.

포함 가능 내용:

- Cursor 오인식
- markdown 오류
- 구조 충돌
- Prompt 충돌
- 문서 누락

---

# 현재 금지 사항

현재 금지:

- 문서 없는 구현
- TASK 없는 구현
- 검증 없는 완료 처리
- 범위 외 수정
- 미래 기능 조기 구현
- 구조 임의 변경

---

# 현재 원칙

현재 프로젝트는 다음 원칙을 유지한다.

- 문서 우선 개발
- 작은 TASK 기반 개발
- AI 협업 가능한 구조 유지
- Prompt 단순화 유지
- 반복 가능한 작업 구조 유지
