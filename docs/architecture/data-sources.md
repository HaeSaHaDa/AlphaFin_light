# AlphaFin LTE 데이터 소스

## 문서 목적

이 문서는 AlphaFin LTE 프로젝트에서 사용하는
데이터 소스와 데이터 수집 범위를 정의한다.

목표는 다음과 같다.

- 데이터 출처 명확화
- 수집 범위 정의
- 데이터 한계 정의
- AI 구현 방향 통일
- 법적 및 구조적 위험 최소화

현재 프로젝트는
무료 또는 저비용 데이터 소스를 우선 사용한다.

---

# 현재 데이터 구조 개요

현재 프로젝트는
다음 데이터 흐름을 기준으로 한다.

```text
외부 데이터 소스
→ Collector
→ Raw Data 저장
→ 전처리
→ Processed Data 생성
→ Retrieval 및 분석
```

---

# 현재 사용 데이터 종류

현재 프로젝트는
다음 데이터를 주요 대상으로 사용한다.

| 데이터 종류 | 목적 |
|---|---|
| 주가 데이터 | 시세 분석 |
| 공시 데이터 | 기업 정보 분석 |
| 뉴스 데이터 | 시장 이벤트 분석 |
| 기업 메타데이터 | 검색 보조 |

# 주가 데이터

## 현재 주요 소스

- pykrx

## 목적

한국 주식 시장 시세 데이터 수집.

## 예상 데이터

- 일봉 데이터
- 거래량
- 시가 / 종가
- 고가 / 저가

## 현재 범위

포함:

- 일봉 데이터
- 종목코드 기반 조회
- CSV 또는 DB 저장

제외:

- Tick 데이터
- 실시간 데이터
- 초고빈도 거래 데이터

# pykrx

## 설명

KRX 기반 한국 주식 데이터 라이브러리.

## 현재 사용 목적

- 시세 데이터 수집
- 종목코드 조회
- 시장 데이터 조회

## 장점

- 무료 사용 가능
- 한국 시장 특화
- pandas 연동 편리

## 현재 한계

- 실시간 데이터 제한
- API 안정성 변동 가능
- 장기 대량 요청 시 제한 가능성

## 현재 원칙

- 대량 요청 최소화
- 샘플 기반 테스트 우선
- 요청 실패 로그 기록

# 현재 저장소 선택

현재 프로젝트는 초기 개발 단계에서
MariaDB를 사용한다.

## 현재 사용 목적

- 로컬 개발 환경 구축
- 수집 데이터 저장
- 전처리 데이터 저장
- 실험 결과 저장
- 초기 검증용 관계형 저장소 구성

## 현재 원칙

- 단순한 테이블 구조를 우선한다.
- SQL이 추적 가능하도록 유지한다.
- 과도한 DB 최적화를 우선하지 않는다.
- DB 종속적인 기능 사용을 최소화한다.

---

# 향후 확장 가능 영역

향후 프로젝트 규모가 커질 경우
다음 확장을 검토할 수 있다.

- PostgreSQL
- Supabase
- 원격 DB 환경
- 클라우드 저장 구조

단,
현재 단계에서는
MariaDB 기반 로컬 구조를 우선한다.
