# TASK-049 이벤트 요약 절단 수정 결과

## 문제

- 이벤트 생성 시 evidence body를 120자로 먼저 제한했다.
- 최종 이벤트 요약을 다시 100자로 제한해 문장 중간에서 끊겼다.
- 기존 DB 캐시는 이미 잘린 요약을 계속 반환했다.
- 프런트 이벤트 카드가 요약을 두 줄로 제한했다.

## 수정

- 이벤트 요약 생성 과정의 120자, 100자 절단을 제거했다.
- Runtime context의 전체 merged evidence를 이벤트 생성에 우선 사용한다.
- 기존 캐시 이벤트도 trace 원본 evidence로 요약을 복원한다.
- evidence의 제목, URL, 발행일도 원본 데이터로 보강한다.
- 이벤트 카드의 `line-clamp-2`를 제거해 전체 요약을 표시한다.

## 검증

- trace: `20260609_154829`
- 뉴스 이벤트 기존 요약 길이: 134자
- 수정 후 실제 API 요약 길이: 703자
- 기사 마지막 문장까지 반환됨을 확인했다.
- 네이버 뉴스 원문 URL 복원을 확인했다.
- Python compileall 통과
- Frontend lint 통과
- TypeScript 검사 통과
- Backend health `200`
- Frontend route `200`

## 남은 데이터 품질 문제

일부 과거 공시 이벤트는 원본 disclosure evidence의 제목 자체가 본문 중간 문장으로 저장되어 있다. 이는 화면 절단 문제가 아니라 수집 또는 공시 chunk 생성 단계의 원본 데이터 문제로 별도 점검이 필요하다.
