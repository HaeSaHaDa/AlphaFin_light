# AlphaFin LTE 범위 정의

## 문서 목적

이 문서는 AlphaFin LTE 프로젝트의
현재 구현 범위를 정의한다.

목표는 다음과 같다.

- 구현 범위 명확화
- 과도한 기능 확장 방지
- AI의 범위 오해 방지
- 실험 가능한 수준 유지

이 프로젝트는
원본 AlphaFin 논문의 전체 구조를
완전히 재현하는 것을 목표로 하지 않는다.

---

# 프로젝트 목표

현재 프로젝트의 핵심 목표는 다음과 같다.

- 한국 금융 데이터 수집
- 금융 문서 검색 구조 구축
- RAG 기반 금융 분석 흐름 구축
- LLM 기반 분석 실험
- 반복 가능한 연구 환경 구축
- AI 협업 기반 개발 구조 구축

---

# 현재 구현 대상

## 1. 한국 주식 시세 데이터 수집

포함:

- pykrx 기반 일봉 데이터
- 종목코드 처리
- CSV 또는 DB 저장
- 샘플 기반 검증

현재 제외:

- 실시간 시세
- 초단타 데이터
- Tick 데이터
- 고빈도 거래 데이터

---

## 2. OpenDART 공시 수집

포함:

- 기업 공시 수집
- 재무 데이터 수집
- 공시 원문 저장
- 메타데이터 저장

현재 제외:

- 전체 시장 완전 동기화
- 초대규모 공시 인덱싱
- 실시간 공시 스트리밍

---

## 3. 뉴스 데이터 수집

포함:

- 뉴스 제목 수집
- 뉴스 본문 수집
- 날짜 및 출처 저장
- 샘플 기반 수집

현재 제외:

- 대규모 뉴스 아카이브 구축
- 상용 뉴스 데이터
- 유료 뉴스 API

---

# RAG 범위

## 포함 범위

현재 목표:

- Chunking
- Embedding
- Vector Search
- Metadata 기반 검색
- Context Assembly
- LLM 입력 구성

현재 후보 기술:

- FAISS
- ChromaDB

---

## 제외 범위

현재 제외:

- 대규모 분산 Retrieval 시스템
- 고성능 Reranking 클러스터
- 멀티 노드 벡터 인프라
- 복잡한 Hybrid Search 최적화
